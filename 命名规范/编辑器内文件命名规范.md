# 编辑器内文件命名规范

# 命名

- 使用 PascalCase 风格对**节点**进行命名, 这与内置的节点大小写风格一致.**如果是中文，则不管。**
- 使用 snake_case 风格为**文件夹和文件**命名(除了c#脚本). 这避免了在 Windows 上导出项目时可能出现的大小写敏感问题.C# 脚本是这个规则的一个例外, 因为按照惯例是用类名来对它们命名, 而类名应该是 PascalCase 风格.**如果是中文，则不管。**
- 通常, 将第三方资源放在顶级的 addons/ 文件夹中, 即使它们不是编辑器插件. 这样更加容易跟踪哪些文件是第三方文件. 当然这个规则也有一些例外: 如果你要使用第三方游戏资源创建角色, 将这些资源和角色场景及脚本放在同一文件夹下会更好.

# 文件组织方式

- addons（插件）
- configs（配置表）
  - tables（数据表 CSV）
  - prompts（提示词文本）
- scenes（游戏场景）
  - ui（界面显示UI）
    - xxxx_panel.tscn
- scripts（代码）
  - global（全局代码）
  - test（测试代码）
  - ui （UI代码）
    - xxx_panel
- sprites（图片）
  - ui
- audios（音频文件）
  - bgm（背景音乐）
  - sfx（音效）
  - se（音效）
- shaders（着色器）
- vfx（粒子特效）
- animations（动画）
- models（3D模型）
- fonts（字体）
- YanGameFrameWork_Godot（框架代码）
- main.tscn
- icon.svg
- project.godot
